= Configurare le credenziali per il cluster PostgreSQL
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
È necessario configurare le credenziali dell'utente del cluster PostgreSQL per eseguire operazioni di protezione dei dati.

.Prima di iniziare
* Se il cluster PostgreSQL non dispone di credenziali valide configurate, l'icona del lucchetto rosso viene visualizzata solo per le risorse rilevate automaticamente. Se durante una successiva operazione di rilevamento si scopre che le credenziali configurate sono errate o non consentono l'accesso al cluster stesso, l'icona del lucchetto rosso ricomparirà.
* È necessario configurare credenziali valide per il cluster PostgreSQL per poter proteggere la risorsa o aggiungerla a un gruppo di risorse per eseguire operazioni di protezione dei dati.
* L'utente PostgreSQL configurato nelle credenziali deve disporre di privilegi sufficienti per accedere al cluster ed eseguire operazioni di backup e ripristino. Come minimo, l'utente deve avere il ruolo *SUPERUSER* o privilegi equivalenti (come le autorizzazioni *pg_read_all_data*, *pg_write_all_data* e *replication*).
* SnapCenter utilizza le credenziali configurate per connettersi all'istanza PostgreSQL tramite il client *psql* o *libpq*. Assicurarsi che gli strumenti client di PostgreSQL siano installati sull'host del plug-in.



NOTE: Prima di configurare il cluster, è necessario prima aggiungere le credenziali PostgreSQL a SnapCenter.

.Fasi
. Nel riquadro di navigazione a sinistra, selezionare *Risorse* e quindi selezionare SnapCenter Plug-in for PostgreSQL cluster dall'elenco.
. Nella pagina risorse, selezionare il tipo di risorsa dall'elenco *Visualizza*.
. (Facoltativo) Selezionare image:../media/filter_icon.gif["icona del filtro"] e selezionare l'host name.
+
A questo punto è possibile selezionare image:../media/filter_icon.gif["icona del filtro"] per chiudere il riquadro dei filtri.

. Seleziona il cluster, quindi seleziona *Configura credenziali*.
. Selezionare le credenziali che sono state configurate in SnapCenter per l'utente del cluster PostgreSQL. Le credenziali devono includere un nome utente e una password validi con privilegi sufficienti per eseguire operazioni di protezione dei dati.
. Selezionare *OK*.


È possibile modificare la configurazione del cluster dalla pagina Topologia.
